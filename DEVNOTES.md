# Rejected Architecture Decisions

## Rejected

### Using spaces for internal category names

Rejected Example:

* "customer service"

Accepted Example:

* customer_service

### Reason

Underscore formatting:

* improves scalability
* improves maintainability
* improves future modularization
* prevents formatting inconsistencies

Presentation formatting is now separated from internal architecture.
